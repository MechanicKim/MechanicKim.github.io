빠르고 간단하고 강력하기까지 하다는 블로그 프레임워크 ‘Hexo’를 이용하여 블로그를 만들면서 겪은 시행착오를 정리해봅니다.

Hello, Hexo!

첫 설치부터 삐걱거림

설치는 매우 간단합니다. 우선 node와 npm을 업데이트 해보도록 하겠습니다. 먼저 n 모듈을 통해 node를 업데이트합니다.

$ node -v
$ sudo npm cache clean -f
$ sudo npm install -g n
$ sudo n stable
$ node -v

그다음 npm을 업데이트합니다.(업데이트에 대한 정보는 velopert님의 블로그를 참고하였습니다.)

$ npm -v
$ sudo npm install -g npm
$ npm -v

그 다음 hexo를 설치합니다.(git은 기본으로 설치 되어있어야 합니다.) 여기서 삐걱거림이 발생했습니다. npm을 통해 설치를 시도하면 긴 에러를 뿜어냅니다.

$ sudo npm install -g hexo-cli

문제는 sudo에 있었습니다. ‘sudo’없이 설치를 해야했던 것이지요. 그러나 ‘sudo’를 없애면 권한문제로 설치가 되지 않습니다. 그렇다면 어떻게 해야할까요? 해답은 다음 이슈 링크에서 찾을 수 있었습니다. 전역으로 node 모듈이 설치되는 위치와 그 안의 모든 폴더 및 파일들의 권한을 바꿔주면 되는 것이지요.

$ sudo chown -R `whoami` /usr/local/lib/node_modules

위 명령이 성공적으로 수행되면 설치도 깔끔하게 끝나게 됩니다.

빠르고 간단하다.

적당한 위치를 잡아 다음 명령을 수행합니다. 초기 구축부터 로컬 구동까지 다음 명령이면 충분합니다.

$ hexo init blog
$ cd blog
$ npm install
$ hexo server
  • 위 명령에서 ‘blog’는 이름으로 변경이 가능합니다.
  • 로컬 서버가 구동되면 ‘http://localhost:4000'로 브라우저에서 접속하여 확인이 가능합니다.

Jekyll이 Ruby기반이라 초기 구축에 어려움이 있었다고 한다면 node 기반의 Hexo는 말 그대로 빠르고 간단하다는 느낌을 받을 수 있었습니다.

테마를 바꾸는 것도 어렵지 않다.

‘themes’ 폴더에 들어가보면 기본 테마인 ‘landscape’가 있는 것을 확인할 수 있습니다. 저 같은 경우는 기본 테마가 성에 차지않아 테마를 찾아보기로 했고 현재 ‘material’ 테마를 적용하게 되었습니다. 테마는 여기서 검색 및 확인이 가능합니다. 물론 ‘material’ 테마도 여기서 찾았습니다.

맘에드는 테마를 찾았다면 ‘themes’폴더에 테마이름의 폴더를 만들고 다운받은 테마를 넣어줍니다. 각 테마의 페이지에서 구축, 설정 가이드를 제공하므로 여기서 자세히 다루지는 않겠습니다. 테마를 넣어준다고 끝나는 것이 아니라 설정파일을 수정해야합니다. _config.yml 파일을 열고 theme 부분을 찾아 변경하고자 하는 테마 이름으로 바꿔주면 됩니다.
