React - 5.Components and Props

Jason Apr 21, 2018
  • React
  • Share to Twitter
  • Share to Facebook
  • Share to Google+

컴포넌트는 UI를 독립적이고 분리를 통해 재사용이 쉽도록 분리하도록 합니다.

개념적으로 컴포넌트는 자바스크립트의 함수와 같습니다. 임의의 입력(props)을 받아 화면에 보여질 React 요소를 결과로 반환합니다.

함수 또는 클래스 컴포넌트

컴포넌트를 정의하는 가장 쉬운 방법은 함수를 사용하는 것입니다.

function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

함수 컴포넌트는 속성(props) 객체를 인자로 받아 React 요소를 반환합니다. 컴포넌트를 정의하는 또 하나의 방법은 ES6 class를 사용하는 것입니다.

class Welcome extends React.Component {
  render() {
    return <h1>Hello, {this.props.name}</h1>;
  }
}

위의 두 컴포넌트는 서로 동일합니다. 클래스를 통한 컴포넌트 정의는 추가적인 특징을 가지지만 다음 포스팅에서 다루기로 하고 이번 포스팅에서는 함수를 통한 컴포넌트 정의에 대해 다루겠습니다.

컴포넌트 그리기

const element1 = <div />;
const element2 = <Welcome name="Sara" />;

위 예제의 element2와 같이 사용자 정의 컴포넌트를 React 요소로 정의할 수 있습니다. 그리고 속성 name(props)을 정의하여 전달할 수 있습니다. 다음 예제는 Welcome 컴포넌트를 정의한 코드입니다.

function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

const element = <Welcome name="Sara" />;
ReactDOM.render(
  element,
  document.getElementById('root')
);

위 예제를 실행하면, 브라우저에 ‘Hello, Sara’가 보여질 것입니다.

  1. ReactDOM.render를 통해 <Welcome name="Sara" /> 그리기를 요청합니다.
  2. React는 {name: 'Sara'}를 속성으로 가지는 Welcome 컴포넌트 호출합니다.
  3. Welcome 컴포넌트가 <h1>Hello, Sara</h1>를 결과로 반환합니다.
  4. <h1>Hello, Sara</h1>를 화면에 그립니다.

컴포넌트를 정의할 때 이름의 첫 글자를 대문자로 시작합니다. 소문자로 시작는 HTML DOM요소와 구분하며 또한 소문자로 시작하는 컴포넌트를 React는 DOM 태그로 취급합니다.

컴포넌트 조합

컴포넌트를 조합하여 하나의 컴포넌트를 만들 수 있습니다. 다음 예제를 보겠습니다.

function Welcome(props) {
  return <h1>Hello, {props.name}</h1>;
}

function App() {
  return (
    <div>
      <Welcome name="Sara" />
      <Welcome name="Cahal" />
      <Welcome name="Edite" />
    </div>
  );
}

ReactDOM.render(
  <App />,
  document.getElementById('root')
);

하나의 App 컴포넌트가 3개의 Welcome 컴포넌트로 이루어져 있습니다.

컴포넌트 추출

컴포넌트를 작게 나누는 것을 두려워하지 마세요. 다음 예제의 Comment 컴포넌트를 보겠습니다.

function Comment(props) {
  return (
    <div className="Comment">
      <div className="UserInfo">
        <img className="Avatar"
          src={props.author.avatarUrl}
          alt={props.author.name}
        />
        <div className="UserInfo-name">
          {props.author.name}
        </div>
      </div>
      <div className="Comment-text">
        {props.text}
      </div>
      <div className="Comment-date">
        {formatDate(props.date)}
      </div>
    </div>
  );
}

위 예제에서 Comment 컴포넌트는 덩어리가 너무 커서 따로 컴포넌트로 분리시킬 수 있는지 살펴보겠습니다. className이 Avatar인 부분을 컴포넌트로 분리할 수 있을 것 같습니다.

function Avatar(props) {
  return (
    <img className="Avatar"
      src={props.user.avatarUrl}
      alt={props.user.name}
    />
  );
}

Avatar 컴포넌트의 속성 부분을 보면 author를 user로 바꾸었습니다. 이 컴포넌트가 한 곳에서만 쓰일 것이 아니라 필요한 곳에 어디든 재사용이 가능하게 되므로 user로 바꾼 것입니다. Comment 컴포넌트에서 Avatar 컴포넌트로 교체하겠습니다.

function Comment(props) {
  return (
    <div className="Comment">
      <div className="UserInfo">
        <Avatar user={props.author} />
        <div className="UserInfo-name">
          {props.author.name}
        </div>
      </div>
      <div className="Comment-text">
        {props.text}
      </div>
      <div className="Comment-date">
        {formatDate(props.date)}
      </div>
    </div>
  );
}

다음은 UserInfo를 컴포넌트로 만들어 보겠습니다. 이 컴포넌트는 Avatar 컴포넌트를 포함합니다.

function UserInfo(props) {
  return (
    <div className="UserInfo">
      <Avatar user={props.user} />
      <div className="UserInfo-name">
        {props.user.name}
      </div>
    </div>
  );
}

역시 author를 user로 바꾸었습니다. Comment 컴포넌트를 수정해 보겠습니다.

function Comment(props) {
  return (
    <div className="Comment">
      <UserInfo user={props.author} />
      <div className="Comment-text">
        {props.text}
      </div>
      <div className="Comment-date">
        {formatDate(props.date)}
      </div>
    </div>
  );
}

이렇게 재사용이 고려되는 부분이나 복잡한 부분들을 컴포넌트로 분리하여 컴포넌트의 크기를 줄이고 단순화 시킬 수 있습니다.

속성은 읽기전용

함수가 받아들인 인자를 변경하려는 시도가 없어 같은 입력에는 항상 같은 결과를 돌려주는 것을 순수 함수라고 합니다. 다음 sum 함수를 예로 들 수 있습니다.

function sum(a, b) {
  return a + b;
}

반대로 순수하지 않음 함수의 예는 다음과 같습니다.

function withdraw(account, amount) {
  account.total -= amount;
}

React는 꽤 유연하지만 순수 함수와 같은 규칙을 가지기도 합니다.

모든 React 컴포넌트의 속성(props)은 순수 함수와 같은 규칙을 가진다.

물론 어플리케이션의 UI가 역동적이고 시간에 따른 변화가 있기 때문에 예외가 필요한데, React는 ‘state’라는 개념을 통해 이를 해결합니다. ‘state’는 다음 포스팅에서 알아보도록 하겠습니다.
