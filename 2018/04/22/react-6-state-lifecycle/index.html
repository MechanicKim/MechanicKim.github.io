상태 변화에 따른 업데이트를 위한 방법

우리가 앞전에 다뤘던 초마다 시간을 찍는 예제를 다시 보겠습니다.

function tick() {
  const element = (
    <div>
      <h1>Hello, world!</h1>
      <h2>It is {new Date().toLocaleTimeString()}.</h2>
    </div>
  );
  ReactDOM.render(
    element,
    document.getElementById('root')
  );
}

setInterval(tick, 1000);

이제 재사용이 가능한 Clock 컴포넌트를 만들고 이 컴포넌트가 자체적으로 타이머를 가지도록 해보겠습니다.

function Clock(props) {
  return (
    <div>
      <h1>Hello, world!</h1>
      <h2>It is {props.date.toLocaleTimeString()}.</h2>
    </div>
  );
}

function tick() {
  ReactDOM.render(
    <Clock date={new Date()} />,
    document.getElementById('root')
  );
}

setInterval(tick, 1000);

기존 예제에서 Clock 컴포넌트(함수형)를 만들고 date를 속성으로 줍니다. 하지만 자체 타이머를 가지는 컴포넌트를 만들기 위해 date를 속성으로 가질 필요가 없을겁니다. 그래서 속성을 아래와 같이 제거해야 합니다.

ReactDOM.render(
  <Clock />,
  document.getElementById('root')
);

자체 타이머를 가지기 위해 state가 필요할 때가 왔습니다. state는 컴포넌트 안에서만 사용되는 private한 속성을 가집니다. 그런데 함수형 컴포넌트에서는 state를 사용할 수 없습니다. 따라서 클래스형 컴포넌트가 필요합니다.

함수형을 클래스형으로 바꾸기

진행 과정을 단계별로 적어보겠습니다.

  1. React.Component를 확장하는 Clock 클래스를 만듭니다.
  2. render 메서드를 정의하고 기존 함수형 컴포넌트의 내용을 render 메서드로 옮깁니다.
  3. 기존 prop를 this.prop로 변경합니다.
  4. 당연히 빈 함수형 컴포넌트는 지웁니다.

바꾼 결과는 다음과 같습니다.

class Clock extends React.Component {
  render() {
    return (
      <div>
        <h1>Hello, world!</h1>
        <h2>It is {this.props.date.toLocaleTimeString()}.</h2>
      </div>
    );
  }
}

이제 클래스 컴포넌트를 통해 local state와 lifecycle hook을 사용할 수 있습니다.

클래스에 Local state 추가하기

클래스에 생성자 메서드인 constructor를 정의할 수 있습니다. 여기에서 state 객체를 초기화하도록 합니다. date를 여기서 초기화합니다. 그리고 render 메서드에서 props 부분을 state로 바꿉니다. 자! 이제 ReactDOM.render 함수에서 Clock 컴포넌트의 date 속성을 지우면 됩니다.

class Clock extends React.Component {
  constructor(props) {
    super(props);
    this.state = {date: new Date()};
  }

  render() {
    return (
      <div>
        <h1>Hello, world!</h1>
        <h2>It is {this.state.date.toLocaleTimeString()}.</h2>
      </div>
    );
  }
}

ReactDOM.render(
  <Clock />,
  document.getElementById('root')
);

이제 남은 것은 자체 타이머를 내장시키는 것입니다.

클래스에 Lifecycle 메서드를 추가하기

컴포넌트가 소멸될 때 자원을 해제시키는 일을 React에서는 어떻게 할까요? 우선 타이머를 셋팅하기로 합니다. Clock 컴포넌트가 다 그려지면 바로 작동하도록 하려고 합니다. 물론 컴포넌트가 제거되기 전 타이머는 해제되어야 겠지요. 다 그려지는 시점과 제거되기 전 시점을 React는 각각 mount와 unmount라고 하며 클래스 컴포넌트에서 메서드로 정의합니다.

class Clock extends React.Component {
  constructor(props) {
    super(props);
    this.state = {date: new Date()};
  }

  componentDidMount() {

  }

  componentWillUnmount() {

  }

  render() {
    return (
      <div>
        <h1>Hello, world!</h1>
        <h2>It is {this.state.date.toLocaleTimeString()}.</h2>
      </div>
    );
  }
}
  1. ReactDOM.render()함수에 <Clock />을 전달할 때, React는 Clock 컴포넌트의 constructor 메서드를 호출합니다. 그리고 해당 시점의 시간 객체를 가지도록 this.state를 초기화합니다.
  2. render 메서드는 최종 결과를 화면에 그려줍니다.
  3. Clock 컴포넌트의 최종 결과가 그려지면 componentDidMount 메서드를 호출합니다. 여기서 tick 메서드를 매초마다 호출하는 타이머를 정의합니다.
  4. tick 메서드에서는 setState 함수를 통해 date값을 업데이트합니다. 여기서 상태변화를 감지한 React는 render 메서드를 호출합니다. 그리고 업데이트된 date값이 반영되여 UI업데이트가 이뤄집니다.
  5. Clock 컴포넌트가 제거될 때 componentWillUnmount 메서드가 호출되고
    타이머는 해제됩니다.

완성된 코드는 다음과 같습니다.

class Clock extends React.Component {
  constructor(props) {
    super(props);
    this.state = {date: new Date()};
  }

  componentDidMount() {
    this.timerID = setInterval(
      () => this.tick(),
      1000
    );
  }

  componentWillUnmount() {
    clearInterval(this.timerID);
  }

  tick() {
    this.setState({
      date: new Date()
    });
  }

  render() {
    return (
      <div>
        <h1>Hello, world!</h1>
        <h2>It is {this.state.date.toLocaleTimeString()}.</h2>
      </div>
    );
  }
}

ReactDOM.render(
  <Clock />,
  document.getElementById('root')
);

state의 올바른 사용법

setState에 대해 알아야 할 3가지가 있습니다.

첫째! 직접 state를 수정하지 마세요!

// Wrong
this.state.comment = 'Hello';

// Correct
this.setState({comment: 'Hello'});

위 예제에서 컴포넌트의 업데이트를 위해서는 setState 함수를 사용해야 함을 말해줍니다. 오직 constructor 메서드에서만 값을 할당하는데 직접 사용합니다.

둘째! state 업데이트는 비동기로 이루어집니다.

// Wrong
this.setState({
  counter: this.state.counter + this.props.increment,
});

// Correct
this.setState((prevState, props) => ({
  counter: prevState.counter + props.increment
}));

// Correct
this.setState(function(prevState, props) {
  return {
    counter: prevState.counter + props.increment
  };
});

위 예제에서 this.state와 this.props 업데이트의 비동기적 특성 때문에 잘못된 예의 경우 의도대로 업데이트가 되지 않을 수 있습니다. setState의 인자를 함수로 주어 이를 해결할 수 있습니다. 이 함수가 이전 상태와 속성 값을 인자로 받아오고 이를 통해 의도대로 업데이트를 할 수 있습니다. 함수는 arrow function과 일반 함수 문법을 모두 사용 가능합니다.

셋째! state에 여러 변수가 있어도 setState 함수를 통해 독립적인 업데이트가 됩니다.

  ...

  constructor(props) {
    super(props);
    this.state = {
      posts: [],
      comments: []
    };
  }

  componentDidMount() {
    fetchPosts().then(response => {
      this.setState({
        posts: response.posts
      });
    });

    fetchComments().then(response => {
      this.setState({
        comments: response.comments
      });
    });
  }

  ...

위 예제에서 constructor 메서드에서 state에 여러 변수를 할당했으면, setState에서 독립적으로 업데이트를 할때는 어떻게 할지 걱정할 필요가 없습니다. setState 함수에서 인자로 넣은 변수만 업데이트되며 나머지는 그대로 유지됩니다.

데이터 흐름

React는 top-down 또는 단방향의 데이터 흐름을 가집니다. 부모 컴포넌트의 props 또는 state 객체의 일부가 자식 컴포넌트의 속성으로 할당되고 자식 컴포넌트는 자신의 속성이 부모의 props 객체에서 온 것인지 state 객체에서 온 것인지 모릅니다. 여기서 알 수 있는 것은 state는 해당 컴포넌트 안에서만 사용되며 다른 컴포넌트에서 접근할 수 없는 것으로 local 또는 캡슐화 되었다고 할 수 있는 것입니다.

function App() {
  return (
    <div>
      <Clock />
      <Clock />
      <Clock />
    </div>
  );
}

ReactDOM.render(
  <App />,
  document.getElementById('root')
);

위 예제 코드에서 Clock 컴포넌트들은 각각 자신만의 타이머를 가지고 독립적으로 업데이트를 합니다.
