문자열도 HTML도 아닌 재미있는 문법

JSX를 소개합니다.

Hello world 예제에서 봤던 <App />가 바로 JSX 문법입니다.

const element = <h1>Hello, world!</h1>;

UI가 어떻게 그려질지를 정의하는 JSX를 React에서는 권장하고 있는데요. 그 이유가 무엇일까요?

Why JSX?

마크업(HTML)과 로직(JavaScript)을 서로 다른 파일로 분리하는 전통적인 방식에서 React는 더 나은 방법을 찾게되는데 핵심은 이것입니다.

UI 렌더링 로직은 이벤트 핸들링, 상태변화, 보여질 데이터와 본질적으로 연결되어 있다.

마크업과 로직이 서로 함께하는 컴포넌트 단위로 구성하는 것이 바로 그것입니다.

JSX안에 자바스크립트 표현식을 넣어보자!

‘{}’를 사용하여 자바스크립트 표현식을 넣을 수 있습니다.

  • 2 + 2
  • user.firstName
  • formatName(user)

아래는 예제 코드입니다.

function formatName(user) {
  return user.firstName + ' ' + user.lastName;
}

const user = {
  firstName: 'Harper',
  lastName: 'Perez'
};

const element = (
  <h1>
    Hello, {formatName(user)}!
  </h1>
);

ReactDOM.render(
  element,
  document.getElementById('root')
);

가독성을 위해 JSX를 줄바꿈과 들여쓰기를 통해 작성합니다. 요구사항은 아니지만 자동 세미콜론 삽입의 문제를 피하기 위해 ‘()’를 사용하여 감싸기를 권장합니다.

JSX도 표현식이다?

컴파일이 끝나면 JSX는 일반 자바스크립트 함수 호출이되고 자바스크립트의 객체가 됩니다. 이것이 의미하는 것은

  1. if 구문이나 for 구문에 JSX를 사용할 수 있고
  2. 변수 할당, 인자로 받아들이고 함수에서 결과로 반환할 수도 있다는 것입니다.
function getGreeting(user) {
  if (user) {
    return <h1>Hello, {formatName(user)}!</h1>;
  }
  return <h1>Hello, Stranger.</h1>;
}

JSX는 속성을 가질 수 있다?

네! 속성을 정의할 수 있고 ‘{}’를 사용하여 자바스크립트 표현식을 넣을 수도 있습니다. 다음과 같이 말이죠.

const element1 = <div tabIndex="0"></div>;
const element2 = <img src={user.avatarUrl}></img>;

참고로 JSX의 속성은 camelCase로 작성합니다.

JSX의 자식을 정의해보자!

JSX도 HTML처럼 자식 요소를 정의할 수 있습니다. 자식 요소가 없다면 />로 마무리 합니다.

const element1 = <img src={user.avatarUrl} />;
const element2 = const element = (
  <div>
    <h1>Hello!</h1>
    <h2>Good to see you here.</h2>
  </div>
);

인젝션 공격 방지

인젝션 공격 방지의 위해

JSX 내에 있는 자바스크립트 변수들은 기본으로 escape 처리됩니다.

이렇게 하여 XSS 공격의 취약점을 보완할 수 있습니다.

JSX는 객체다!

Babel이 JSX(element1)를 컴파일하면 객체(element2)가 됩니다. 아래 예제의 element1과 element2는 서로 동일합니다.

const element1 = (
  <h1 className="greeting">
    Hello, world!
  </h1>
);

const element2 = React.createElement(
  'h1',
  {className: 'greeting'},
  'Hello, world!'
);

React.createElement 가 객체를 생성하며 이 객체를 React elements라고도 부릅니다. React.createElement는 버그없는 코드를 작성하도록 돕는 몇가지 검사를 수행하 다음과 같은 객체를 만들어 냅니다.

// Note: this structure is simplified
const element3 = {
  type: 'h1',
  props: {
    className: 'greeting',
    children: 'Hello, world!'
  }
};

React는 이 객체들을 읽어들여 DOM을 구성하고 최신으로 유지합니다. DOM에 요소들을 그리는 내용은 다음 포스팅에서 계속 다뤄보겠습니다.
